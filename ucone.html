Plano de Curso: Analisar Requisitos e Funcionalidades da Aplicação

Carga Horária: 88 horas

Objetivo Geral

Capacitar os alunos a realizarem a análise de requisitos e funcionalidades de aplicações, identificando necessidades dos usuários e estruturando modelos para o desenvolvimento de software.

Estrutura do Curso

- Conceitos fundamentais de Engenharia de Software
- Diferença entre requisitos funcionais e não funcionais
- Ciclo de vida do desenvolvimento de software

- Métodos de coleta de requisitos (entrevistas, observação, questionários)
- Identificação de stakeholders
- Técnicas de brainstorming e dinâmicas para elicitação de requisitos

- Estruturação de um briefing de projeto
- Elaboração de especificações funcionais e não funcionais
- Introdução ao UML: Casos de Uso e Diagramas de Classes
- Ferramentas para documentação de requisitos (Notion, Trello, Jira, Figma)

- Princípios de organização de conteúdo
- Estruturas de navegação e experiência do usuário (UX)
- Prototipação de telas (baixa fidelidade)

- Testes de usabilidade
- Simulação de cenários com base nos requisitos levantados
- Reunião de validação com stakeholders

- Desenvolvimento de um projeto real baseado em um caso de estudo
- Aplicação de todas as etapas aprendidas
- Apresentação dos projetos para avaliação final

Metodologia

  • Aulas expositivas e interativas
  • Estudos de caso
  • Atividades práticas com levantamento e análise de requisitos reais
  • Uso de ferramentas digitais para modelagem e documentação
  • Trabalho em equipe para simular ambiente de TI corporativo

Avaliação

  • Participação e exercícios individuais (20%)
  • Atividades práticas em grupo (30%)
  • Projeto final de análise de requisitos (50%)
